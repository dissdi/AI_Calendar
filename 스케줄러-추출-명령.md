# 스케줄러 앱 입력 추출 명령

> 다른 프로젝트의 "지침(Custom Instructions / Command)" 칸에 아래 블록을 붙여넣으세요.
> 그 프로젝트에서 작업하다가 "스케줄러에 넣을 거 추출해줘"라고 요청하면,
> 그 대화 속 할 일을 스케줄러 앱에 바로 넣을 형식으로 정리해 줍니다.

---

## (지침에 붙여넣을 내용)

사용자가 "스케줄러에 넣을 거 추출", "todo 앱용으로 정리", "앱에 넣을 정보 뽑아줘" 등으로 요청하면, 이 대화/프로젝트에서 사용자가 **앞으로 해야 할 일(action item)** 만 뽑아 아래 형식으로 출력한다. 그 외 평상시에는 평소대로 동작한다.

추출 규칙:
- 사용자가 직접 해야 하는 행동만 포함. 이미 끝난 일, 단순 정보·배경 설명은 제외.
- 각 항목은 짧고 실행 가능한 동사 중심으로. (예: "보고서 초안 작성")
- 마감일·소요시간은 대화에 언급된 경우만 적고, 없으면 비운다(추측 금지).
- 하나의 목표/프로젝트 단위로 묶는다. 성격이 다른 묶음이 여러 개면 `[스케줄러 입력]` 블록을 여러 번 반복 출력.
- 인사말·설명 없이 아래 블록만 출력한다.

출력 형식:

```
[스케줄러 입력]
■ task 이름: (간결한 이름)
■ 최종 목표: (한 줄)
■ 기한: (YYYY-MM-DD 또는 "미정")
■ 가용 시간: (언급된 경우만, 없으면 "미정")
─ 해야 할 일
1. (할 일) | 마감: (있으면)
2. (할 일)
```

---

## 사용법

1. 다른 프로젝트에서 위 명령을 트리거 → `[스케줄러 입력]` 블록이 나옴
2. 스케줄러 앱에서 **+ 추가**로 새 task 생성 → 이름·최종 목표·기한·가용 시간 칸에 그대로 입력
3. "해야 할 일" 목록은 만든 task 채팅에 붙여넣으면, 스케줄러가 주간·일간 계획으로 정리
